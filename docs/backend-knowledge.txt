================================================================================
TỔNG HỢP KIẾN THỨC & FLOW - QUANDH-CORE BACKEND
================================================================================

1. TỔNG QUAN DỰ ÁN
================================================================================

- Framework: Laravel 12, PHP 8.4
- Database: MySQL
- Môi trường: Laravel Sail (Docker)
- Auth: Laravel Sanctum (Bearer token)
- Phân quyền: Spatie Laravel Permission (teams mode, organization-based)
- Media: Spatie Laravel Media Library
- Import/Export: Maatwebsite Excel
- API Docs: Knuckles Scribe
- Ngôn ngữ: Tiếng Việt (comment, docs, label, response message)

5. FLOW XỬ LÝ REQUEST
================================================================================

5.1 MIDDLEWARE CHAIN (Protected routes):
─────────────────────────────────────────
Request → auth:sanctum → set.permissions.team → log.activity → Controller
          │                │                     │
          │                │                     └─ Ghi nhật ký vào DB
          │                │
          │                └─ Đọc X-Organization-Id header
          │                   → Check org tồn tại + active
          │                   → Check user có role trong org
          │                   → setPermissionsTeamId(orgId)
          │
          └─ Verify Bearer token → set Auth::user()

Trên từng route còn có:
  →  permission:{resource}.{action},web  (Spatie middleware)

5.2 CONTROLLER FLOW:
─────────────────────
  FormRequest validate → Service method → RespondsWithJson response

5.3 SERVICE FLOW:
─────────────────
  - Read operations: query + filter + return data
  - Write operations: DB::transaction() nếu >= 2 bước
  - Media: qua MediaService (không dùng Storage trực tiếp)

5.4 RESPONSE FORMAT (RespondsWithJson Trait):
─────────────────────────────────────────────
  success($data, $message)              → stats, destroy, bulk, import
  successResource(Resource, $message)   → show, store(201), update, changeStatus
  successCollection(Collection, $msg)   → index, tree
  error($message, $code, $errors)       → lỗi chung
  unauthorized() / forbidden() / notFound() / conflict() → shortcuts

Response structure:
  { success: true/false, message: "...", data: {...}, meta: {pagination} }

6. AUTHENTICATION FLOW
================================================================================

6.1 LOGIN:
──────────
POST /api/auth/login { email, password }

  → AuthService.login()
    → Tìm user theo email hoặc user_name
    → Check password (Hash::check)
    → Check status (active)
    → createToken('auth_token') → plainTextToken
    → getAccessibleOrganizations(user) → danh sách org user có role
    → getRolesAndPermissionsForOrganization(user, orgId)
      → setPermissionsTeamId(orgId)
      → user.getAllPermissions() → pluck names
      → CaslAbilityConverter::toCaslAbilities(permissions)

Response:
  {
    access_token: "1|abc...",
    token_type: "Bearer",
    user: { id, name, email, status, ... },
    available_organizations: [{ id, name, description }],
    current_organization_id: 2,
    roles: ["admin"],
    permissions: ["users.index", "posts.store", ...],
    abilities: [{ action: "index", subject: "User" }, ...]
  }

6.2 SWITCH ORGANIZATION:
─────────────────────────
POST /api/auth/switch-organization { organization_id }
Headers: Authorization: Bearer {token}

  → Check org hợp lệ + user có quyền
  → Trả về roles, permissions, abilities mới cho org đó

6.3 CASL ABILITY CONVERTER:
────────────────────────────
Permission "posts.index" → { action: "index", subject: "Post" }
Permission "post-categories.tree" → { action: "tree", subject: "PostCategory" }
Permission "issuing-agencies.store" → { action: "store", subject: "IssuingAgency" }

Quy tắc: resource name → PascalCase (bỏ dấu gạch, viết hoa chữ đầu)

7. PHÂN QUYỀN (PERMISSION SYSTEM)
================================================================================

7.1 CẤU TRÚC:
──────────────
- Spatie Permission với Teams mode ON
- Team column: organization_id
- Guard: web (dùng chung cho web và API Sanctum)
- User model: $guard_name = 'web'

7.2 FORMAT PERMISSION:
──────────────────────
{resource}.{action}
Ví dụ: posts.index, users.store, documents.export

7.3 FLOW KIỂM TRA QUYỀN:
──────────────────────────
1. SetPermissionsTeamId middleware: set organization context
2. Route middleware: permission:posts.index,web
3. Spatie kiểm tra: user → roles (trong org) → role_has_permissions → có permission?

7.4 BẢNG LIÊN QUAN:
────────────────────
permissions              → Danh sách quyền (name, guard, description, parent_id)
roles                    → Vai trò (name, guard, organization_id=null sau globalize)
role_has_permissions     → Role ↔ Permission
model_has_roles          → User ↔ Role (có organization_id)
model_has_permissions    → User ↔ Permission trực tiếp (hiện không dùng)

7.5 ROLES MẶC ĐỊNH:
────────────────────
- Super Admin: toàn bộ permissions
- Admin: toàn bộ permissions
- Editor: posts.*, post-categories.*
- Vai trò mẫu: chỉ xem (stats, index, show)

7.6 PERMISSION LIST:
────────────────────
users:            stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
permissions:      stats, index, tree, show, store, update, destroy, bulkDestroy, export, import
roles:            stats, index, show, store, update, destroy, bulkDestroy, export, import
organizations:    stats, index, tree, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
posts:            stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import, incrementView
post-categories:  stats, index, tree, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
log-activities:   stats, index, show, export, destroy, bulkDestroy, destroyByDate, destroyAll
documents:        stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
document-types:   stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
issuing-agencies: stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
issuing-levels:   stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import
document-signers: stats, index, show, store, update, destroy, bulkDestroy, bulkUpdateStatus, changeStatus, export, import

9. API ROUTES
================================================================================

9.1 PUBLIC (không cần auth):
────────────────────────────
POST   /api/auth/login
POST   /api/auth/forgot-password
POST   /api/auth/reset-password
GET    /api/settings/public
GET    /api/{resource}/public            → Dữ liệu công khai đầy đủ
GET    /api/{resource}/public-options    → Dropdown tối giản (id, name, description)

Resources có public: post-categories, organizations, document-types, document-signers,
                     document-fields, issuing-agencies, issuing-levels

9.2 PROTECTED (cần Bearer token + X-Organization-Id):
──────────────────────────────────────────────────────
POST   /api/auth/logout
POST   /api/auth/switch-organization
GET    /api/user                         → Thông tin user hiện tại

Mỗi resource có pattern routes:
GET    /api/{resource}/stats             → permission:{resource}.stats
GET    /api/{resource}/export            → permission:{resource}.export
POST   /api/{resource}/import            → permission:{resource}.import
POST   /api/{resource}/bulk-delete       → permission:{resource}.bulkDestroy
PATCH  /api/{resource}/bulk-status       → permission:{resource}.bulkUpdateStatus
GET    /api/{resource}                   → permission:{resource}.index
POST   /api/{resource}                   → permission:{resource}.store
GET    /api/{resource}/{id}              → permission:{resource}.show
PUT    /api/{resource}/{id}              → permission:{resource}.update
DELETE /api/{resource}/{id}              → permission:{resource}.destroy
PATCH  /api/{resource}/{id}/status       → permission:{resource}.changeStatus

Đặc biệt:
GET    /api/{resource}/tree              → Cấu trúc cây (organizations, permissions, post-categories)
POST   /api/posts/{post}/view            → Tăng lượt xem

11. LOG ACTIVITY MIDDLEWARE
================================================================================

File: app/Modules/Core/Middleware/LogActivity.php

Ghi nhật ký mọi request API vào bảng log_activities.

Thông tin ghi: description, user_type, user_id, organization_id, route,
              method_type, status_code, ip_address, country, user_agent, request_data

Trường nhạy cảm bị loại trừ:
  password, password_confirmation, _token, token,
  email_smtp_password, sms_password, zalo_password, chat_api_key,
  api_gemini_token, api_deepseek_token, api_chatgpt_token,
  api_firebase_token, api_google_maps_token

Path không log: /up (health check)

Description format: "{Action} {resource}" hoặc "{Action} {resource} #{id}"
Ví dụ: "Truy cập danh sách bài viết", "Xem chi tiết người dùng #5"

Resource labels:
  users=người dùng, posts=bài viết, post-categories=danh mục bài viết,
  permissions=quyền, roles=vai trò, organizations=tổ chức,
  documents=văn bản, document-types=loại văn bản,
  issuing-agencies=cơ quan ban hành, issuing-levels=cấp ban hành,
  document-signers=người ký, document-fields=lĩnh vực,
  settings=cấu hình hệ thống, log-activities=nhật ký truy cập

Khi thêm module mới → CẬP NHẬT: resourceLabel(), actionLabels, pathActions, route params

12. EXPORT / IMPORT PATTERN
================================================================================

12.1 EXPORT:
────────────
- Class: implements FromCollection, WithHeadings
- Constructor nhận filters
- collection(): query với filter + load relations + map fields
- headings(): tiếng Việt
- Export phải giống Resource (cùng fields)
- Các trường quan hệ (created_by, categories...) phải có

Ví dụ PostsExport:
  Headings: ID, Tiêu đề, Slug, Nội dung, Trạng thái, Lượt xem, Danh mục, Người tạo, Người sửa, Ngày tạo, Ngày cập nhật
  Categories: nối bằng dấu phẩy

12.2 IMPORT:
────────────
- Class: implements ToModel, WithHeadingRow
- Validate: file required|mimes:xlsx,xls,csv|max:10240
- Scribe PHPDoc: ghi rõ cột bắt buộc, cột optional + giá trị mặc định
